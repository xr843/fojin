## V2EX (分享创造 / 程序员)

**标题**: 开源了一个聚合全球 440+ 个佛教古籍数据库的平台，支持 30 语种 AI 问答

**正文**:

分享一个我做的开源项目 —— 佛津 (FoJin)。

做这个项目的起因是：全球佛教文本散落在几百个数据库中（CBETA、SuttaCentral、BDRC、SAT、84000……），每个数据库有不同的界面、搜索方式和数据格式。研究者花在"找经文"上的时间比"读经文"还多。

佛津把 440+ 个数据源聚合到一个平台，主要功能：

- **联合检索**：本地 ES 索引 + 典津（清华/知网）72.8 万条跨平台联检
- **全文阅读**：4,488 卷经文在线阅读
- **多语种对读**：30 语种平行对照（汉/梵/巴利/藏/英/日/韩……）
- **辞典检索**：6 部权威辞典、237,593 条词条
- **知识图谱**：9,600+ 实体、3,800+ 关系，力导向图可视化
- **AI 问答**：RAG + 大模型，基于 1,100 万字经文原文回答

技术栈：FastAPI (async) + React 18 + TypeScript + PostgreSQL (pgvector) + Elasticsearch 8 + Redis + Docker Compose

一条命令部署：`docker compose up -d`

GitHub: https://github.com/xr843/fojin
协议：Apache 2.0

欢迎 Star、提 Issue、贡献数据源！

### 技术亮点

- **异步全栈**：FastAPI + SQLAlchemy async + httpx AsyncClient，单机支撑 440+ 数据源并发检索
- **向量+全文双引擎**：PostgreSQL pgvector（语义搜索） + Elasticsearch 8 ICU（多语种分词），互为补充
- **19 个数据导入管线**：每个数据源一个独立脚本，统一 import_all.py 编排
- **安全加固**：非 root 容器、多阶段构建、127.0.0.1 端口绑定、资源限制、CSP 安全头
- **RAG AI**：pgvector 语义检索，覆盖 7,290 部经文（34.7 万语义片段）+ 多 LLM provider 支持
