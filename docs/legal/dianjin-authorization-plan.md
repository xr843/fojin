## 重新公开前的检查清单

- [ ] 获得典津团队书面授权（邮件回复即可）
- [ ] 根据授权要求调整代码中的归属标注
- [ ] 在 README 中添加典津使用声明
- [ ] 在 LICENSE 或 NOTICE 文件中明确第三方 API 使用条款
- [ ] 确认 API Key 获取方式的合规性
- [ ] 将仓库重新设为 Public：`gh repo edit xr843/fojin --visibility public`

## 备选方案（如果典津不授权）

如果典津团队不同意在开源项目中集成其 API：

1. 从公开代码中移除所有典津相关代码
2. 将典津功能改为可选插件/扩展，不包含在主仓库中
3. 仅保留佛津自有的 483 数据源 + 本地搜索 + AI 问答等核心功能
4. 这些核心功能完全不依赖典津，项目价值不受实质影响
